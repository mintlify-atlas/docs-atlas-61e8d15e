## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** (v18 or higher)
- **npm**, **yarn**, or **pnpm** package manager
- **Git** for version control
- **Supabase CLI** (optional, for local development)

## Quick Start

### 1. Clone the Repository

```bash
git clone <repository-url>
cd creativa-labs-cabina-de-fotos
```

### 2. Install Dependencies

<CodeGroup>
```bash npm
npm install
```

```bash yarn
yarn install
```

```bash pnpm
pnpm install
```
</CodeGroup>

### 3. Configure Environment Variables

Create a `.env.local` file in the root directory:

```bash
cp .env.example .env.local
```

See [Environment Variables](/technical/environment-variables) for detailed configuration.

### 4. Run Development Server

<CodeGroup>
```bash npm
npm run dev
```

```bash yarn
yarn dev
```

```bash pnpm
pnpm dev
```
</CodeGroup>

The application will start on `https://localhost:3000` with HTTPS enabled.

<Note>
The development server uses a self-signed SSL certificate for local HTTPS. Your browser may show a security warning - this is normal for local development.
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 3000 already in use">
    The app is configured to run on port 3000. If this port is busy, you can:
    
    1. Stop the process using port 3000
    2. Modify the port in `vite.config.ts` under `server.port`
  </Accordion>

  <Accordion title="SSL certificate warnings">
    The development server uses `@vitejs/plugin-basic-ssl` for HTTPS. Browser warnings are expected.
    
    To trust the certificate:
    - Chrome: Click "Advanced" → "Proceed to localhost"
    - Firefox: Click "Advanced" → "Accept the Risk and Continue"
  </Accordion>

  <Accordion title="Supabase connection errors">
    Ensure your `.env.local` contains valid Supabase credentials:
    - `VITE_SUPABASE_URL`
    - `VITE_SUPABASE_ANON_KEY`
    
    Check that your Supabase project is active and accessible.
  </Accordion>
</AccordionGroup>
