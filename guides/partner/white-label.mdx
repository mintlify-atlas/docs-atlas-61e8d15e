## White-Label Overview

Cabina's white-label features allow you to offer AI photo booth services under your own brand. Your clients and their guests see your logo, colors, and styling—not Cabina's.

## What Can Be White-Labeled?

### Visual Identity
- 🎨 **Logo**: Your business logo appears in all events
- 🎨 **Primary Color**: Brand color for buttons, accents, highlights
- 🎨 **Border Radius**: UI element styling (sharp, rounded, premium)
- 🎨 **Welcome Text**: Custom greeting messages

### Content
- ✨ **Style Packs**: Choose which AI styles are available
- 📸 **Gallery Branding**: Your logo in event galleries
- 📄 **QR Codes**: Branded QR code designs (with your logo)

### What's Not White-Labeled
- Small "Powered by MetaLab IA" footer text (required)
- Core app infrastructure

<Info>
**Premium White-Label:** For complete whitelabeling (including removal of "Powered by" text), contact sales about enterprise plans.
</Info>

## Accessing Branding Settings

<Steps>
  <Step title="Navigate to Branding Tab">
    From your Partner Dashboard, click **Branding** in the header navigation.
  </Step>
  
  <Step title="Branding Panel Sections">
    You'll see two main sections:
    - **Identidad Visual** (Visual Identity): Logo, colors, styling
    - **Estilos IA** (AI Styles): Style pack selection
  </Step>
</Steps>

## Configuring Visual Identity

### Upload Your Logo

<Steps>
  <Step title="Click Upload Area">
    In the "Logo Corporativo" section, click the dashed upload box.
  </Step>
  
  <Step title="Select Image File">
    Choose your logo file:
    - **Formats**: PNG (recommended), JPG, WebP
    - **Size**: Max 5MB
    - **Dimensions**: 400x400px recommended (square or horizontal)
    - **Background**: Transparent PNG works best
  </Step>
  
  <Step title="Upload Completes">
    - Logo previews immediately in the box
    - Hover to see Edit2 icon overlay
    - Click again to replace with different logo
  </Step>
</Steps>

**Logo Best Practices:**

✅ **Do:**
- Use transparent background PNG
- Keep logos simple and readable at small sizes
- Test on mobile devices (appears in header at ~40-60px height)
- Use horizontal or square orientation

❌ **Don't:**
- Use overly complex designs (poor visibility when small)
- Include taglines or fine text (won't be readable)
- Use light colors on white (ensure contrast)
- Upload very large files (slows load times)

**Where Logo Appears:**
- Event kiosk interface (top-left header)
- Event organizer dashboard (header)
- QR code downloads (on branded QR designs)
- Email invitations to clients

### Set Primary Color

<Steps>
  <Step title="Click Color Preview Box">
    In the "Color de Acento" section, click the colored square.
  </Step>
  
  <Step title="Use Color Picker">
    - Hidden color input activates
    - Choose your brand's primary color
    - Or enter HEX code directly
  </Step>
  
  <Step title="Preview Updates">
    The color preview updates immediately, showing your selection.
  </Step>
</Steps>

**Color Selection Tips:**

- **High Contrast**: Ensure color is visible on dark backgrounds
- **Accessibility**: Avoid very light colors (poor readability)
- **Brand Consistency**: Match your website/marketing materials
- **Test on Mobile**: Colors may appear different on various screens

**Where Color Applies:**
- Button backgrounds (Generate, Download, etc.)
- Selected style card borders
- Progress bars and loading indicators
- Active state highlights
- QR code accent elements

**Popular Brand Colors:**

| Brand Style | Suggested HEX | Use Case |
|-------------|---------------|----------|
| Tech/Modern | #135bec (Blue) | Corporate, professional |
| Creative/Fun | #ec4899 (Pink) | Weddings, celebrations |
| Luxury/Premium | #7f13ec (Purple) | High-end events |
| Energy/Bold | #ff5500 (Orange) | Active, energetic brands |
| Nature/Organic | #10b981 (Green) | Eco-friendly, wellness |

<Note>
**Default Color:** If no color is set, the default Cabina orange (#ff5500) is used.
</Note>

### Configure Border Radius

<Steps>
  <Step title="Open Radius Dropdown">
    In the "Radio de Bordes" section, click the dropdown menu.
  </Step>
  
  <Step title="Select Style">
    Choose from preset options:
    - **4px (Recto)**: Sharp, modern, minimal
    - **8px (Suave)**: Soft, balanced
    - **12px (Premium)**: Elegant, professional (default)
    - **20px (Redondeado)**: Friendly, playful
  </Step>
</Steps>

**Radius Impact:**

- Affects buttons, cards, inputs, modals
- Influences overall UI aesthetic
- Should match your brand's design language

**Which to Choose:**
- **Sharp (4px)**: Tech companies, modern brands
- **Soft (8px)**: Balanced for most use cases
- **Premium (12px)**: Upscale events, luxury brands
- **Rounded (20px)**: Fun events, family-friendly brands

### Save Visual Identity

After configuring logo, color, and radius:

<Steps>
  <Step title="Click 'Sincronizar Panel'">
    At the bottom of the Identidad Visual section, click the white **"Sincronizar Panel"** button.
  </Step>
  
  <Step title="Wait for Confirmation">
    - Button shows loading spinner
    - Success toast appears: "Branding actualizado"
    - Changes apply to all future events immediately
  </Step>
</Steps>

<Warning>
**Existing Events:** Branding changes do NOT automatically apply to events created before the update. To update an existing event's branding, edit the event and re-save it.
</Warning>

## Selecting AI Style Packs

Control which styles are available to your clients and their event guests.

### Understanding Style Packs

Styles are organized into packs (subcategories):

- **Profesional**: Business portraits, LinkedIn-style
- **Artístico**: Creative, artistic effects
- **Vintage**: Retro, classic, nostalgic looks
- **Fantástico**: Fantasy, dramatic, cinematic
- **Clásico**: Timeless, elegant portraits
- **Custom Packs**: Additional packs based on IDENTITIES

### Enabling/Disabling Packs

<Steps>
  <Step title="Scroll to 'Estilos IA' Section">
    In the Branding tab, find the AI Styles section below Visual Identity.
  </Step>
  
  <Step title="View Style Pack Grid">
    All available packs display as thumbnail cards:
    - Pack name
    - Preview image
    - Checkmark if currently enabled
  </Step>
  
  <Step title="Toggle Selection">
    - Click a pack thumbnail to enable/disable
    - Selected packs show a checkmark overlay
    - De-selected packs are dimmed
  </Step>
  
  <Step title="Save Changes">
    Click **"Sincronizar Panel"** to apply style pack changes.
  </Step>
</Steps>

**Pack Selection Strategy:**

✅ **Enable packs that:**
- Match your target market (corporate vs. creative vs. wedding)
- Your clients frequently request
- Produce high-quality results
- Differentiate your offering

❌ **Disable packs that:**
- Don't fit your brand (e.g., fantasy styles for corporate events)
- Confuse users with too many options
- Have low quality results
- You want to reserve for premium tiers

### Style Pack Recommendations by Market

| Market Segment | Recommended Packs | Notes |
|----------------|-------------------|-------|
| Corporate Events | Profesional, Clásico | Professional, clean looks |
| Weddings | Artístico, Vintage, Fantástico | Romantic, dramatic |
| Birthday Parties | All packs | Variety for fun |
| Brand Activations | Artístico, Custom | Unique, shareable |
| Conferences | Profesional only | Consistent, professional |

<Info>
**Client Override:** When creating events, you can select specific styles per event, even if not all packs are enabled at the partner level.
</Info>

## Client-Level Branding

When you associate an event with a client, that client can customize event-specific branding.

### What Clients Can Customize

**Event Organizer Dashboard Access:**
- Event name
- Welcome text/message
- Event logo (overrides your partner logo)
- Event primary color (overrides your partner color)
- Event date and time

**What Clients Cannot Change:**
- Your partner-level defaults
- Available style packs (set by you)
- Core event settings (slug, credits)
- Other clients' events

### Partner vs. Client Branding Hierarchy

```
Default Branding: Cabina defaults
  ⬇️
Overridden by: Partner-Level Branding (your logo, colors)
  ⬇️
Optionally Overridden by: Client Event-Specific Branding
```

**Example Flow:**
1. You set partner logo to "Your Agency Logo"
2. You create event for Client A
3. Client A accesses their dashboard
4. Client A uploads their own logo
5. Event shows Client A's logo (not yours)
6. Event still uses your color and style packs

<Note>
**Best Practice:** For white-label resale, encourage clients to upload their own logos for a fully customized experience.
</Note>

When you generate QR codes for events, they include your branding.

### QR Code Elements

**Included in Design:**
- Event name (from event settings)
- Your partner logo (from Visual Identity)
- Your primary color (as accent in design)
- QR code itself (linking to event URL)
- "Powered by MetaLab IA" footer

**Customization:**
- Background color (based on your primary color)
- Border styling (matches your radius setting)
- Logo placement (centered at top)

## Troubleshooting

<AccordionGroup>
  <Accordion title="Logo not appearing on events">
    **Checks:**
    - Verify logo uploaded successfully (visible in Branding panel)
    - Ensure you clicked "Sincronizar Panel" to save
    - Check event was created AFTER branding was saved
    - Try re-uploading logo (may be file corruption)
    
    **Solution:**
    - Delete and re-upload logo
    - Use PNG format with transparent background
    - Ensure file size under 5MB
    - Clear browser cache and refresh
  </Accordion>
  
  <Accordion title="Color not applying correctly">
    **Possible Causes:**
    - HEX code entered incorrectly
    - Browser cache showing old color
    - Event created before color was updated
    
    **Solution:**
    - Re-enter HEX code (e.g., #135bec)
    - Save with "Sincronizar Panel"
    - Hard refresh event page (Ctrl+Shift+R)
    - For existing events, edit and re-save them
  </Accordion>
  
  <Accordion title="Client can't customize event branding">
    **Checks:**
    - Event is associated with client account (verify in edit modal)
    - Client has received invitation email
    - Client is signed in with correct email
    - Client has Event Organizer role (not End User)
    
    **Solution:**
    - Edit event and verify client email is correct
    - Resend invitation via Clients tab
    - Have client check spam folder
    - Contact support if invitation not arriving
  </Accordion>
  
  <Accordion title="Style packs not showing for clients">
    **Issue:**
    Clients or event guests see no styles or limited styles.
    
    **Solution:**
    - Verify packs are enabled in Branding > Estilos IA
    - Click packs to select (checkmark overlay appears)
    - Save with "Sincronizar Panel"
    - Edit event and ensure styles are selected in event modal
    - At least one pack must be enabled
  </Accordion>
  
  <Accordion title="Changes not reflecting on existing events">
    **Expected Behavior:**
    Branding changes only apply to NEW events created after saving.
    
    **To Update Existing Events:**
    1. Navigate to Events tab
    2. Click Edit (Edit2 icon) on each event
    3. No changes needed, just click "Guardar Cambios"
    4. Event inherits latest partner branding
    
    **Note:** This must be done manually per event.
  </Accordion>
</AccordionGroup>
